-- PASSO A PASSO (ordem sugerida)
--
-- A) SQL no Supabase (SQL Editor → New query)
--    Cola e executa SÓ o ficheiro:
--    supabase/migrations/20260410190000_mercado_pago_checkout_intents.sql
--
-- B) Credenciais Mercado Pago (conta onde queres receber o dinheiro)
--    1. Abre: https://www.mercadopago.com.br/developers/panel/app
--    2. Credenciais de produção (ou “Teste” para experimentar sem cobrar).
--    3. Copia o **Access Token** (começa por APP_USR-… ou TEST-…).
--    Não precisas da “Public Key” nesta integração: o checkout abre no site
--    do Mercado Pago; o token secreto vai só no Supabase (passo C).
--
-- C) Secrets no Supabase (liga o MP ao teu projeto)
--    1. Supabase Dashboard → teu projeto → **Project Settings** → **Edge Functions** → **Secrets**
--    2. Adiciona:
--       Nome: MERCADOPAGO_ACCESS_TOKEN     Valor: (cola o Access Token do passo B)
--       Nome: APP_PUBLIC_URL               Valor: https://maes-pi.vercel.app
--       (troca o URL se o teu site for outro; tem de ser https e sem barra no fim)
--
-- D) Publicar as funções (obrigatório — sem isto o pagamento dá sempre erro)
--    Opção 1 — duplo clique no Windows (na pasta do repo):
--       supabase\deploy-mp-functions.cmd
--    Opção 2 — PowerShell na pasta MaesAtipicas:
--       npx supabase login
--       npx supabase functions deploy mercadopago-create-preference --project-ref ahjhjzdmkkrcgbuxmhww
--       npx supabase functions deploy mercadopago-webhook --project-ref ahjhjzdmkkrcgbuxmhww
--    (Se o project-ref for outro, vê em Supabase → Settings → API → URL do projeto)
--
-- E) Opcional no código (Vercel / domínio fixo)
--    Em supabase-config.js podes preencher:
--    window.AURA_APP_PUBLIC_URL = 'https://maes-pi.vercel.app';
--    Isto reforça o domínio enviado ao servidor no pagamento.
